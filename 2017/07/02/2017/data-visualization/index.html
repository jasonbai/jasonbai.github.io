数据可视化感悟

发表于 2017-07-02

成功提交

终于算是成功通过了Udacity-数据分析的P6-可视化大作业。结束了将近一个月的折磨。

项目背景

这次课程的主题是可视化，作业开头的导言写的很好。

为什么进行这个项目？
该项目涉及对于有效数据可视化至关重要的总体态度和信念，如：

  • 可视化是一场对话
  • 向别人展示和分享可视化
  • 可视化是一个流动过程，通常需要多次改进迭代

你将有机会体验端对端的有效数据可视化创建过程，突出原本隐藏或很难发现的重要数据信息。

我会学到什么？
完成此项目后，你将能够：

  • 展现出选择最佳视觉元素来编码数据和批判性评估可视化效果的能力
  • 使用交互式可视化向合适的受众讲述故事或发现
  • 经历迭代式可视化创建流程和使用 dimple.js 或 d3.js 建立交互式可视化。

在日常的工作中，没有数据分析师给我提供分析报告，我自己边做分析，边写结论，很多都是用Excel或者RStudio直接出图，接着继续寻找自己感兴趣的东西。没有在意太多需要对外沟通的东西。如题中所说，可视化是一场对话，过去我是自己和自己对话，正因如此，这次作业花了很大的精力去做，与自己对话，与他人对话都是应该去训练的。

这次可视化作业，我没有直接使用D3.js，而是用dimple.js来制作，主要原因：懒。

D3.js偏向底层，有大量的可定制化的细节自由调整。可是除去专业人士需要那样疯狂的调整细节，多数人如我一样，手头就有很好的工具，简单的如excel，高级的Tableau，阿里云提供的datav，都要比纯粹的写D3.js要更快且更有效率。当然，由于作业要求，起码要使用dimple.js这样的封装了部分D3.js代码的工具来完成。

既然最终呈现，是通过网页展示，利用前期学的一点点网站前端的经验，套用了一个html5的模板，进行了css代码的微调，最终呈现出了http://www.jasonbai.com/Udacity-P6 的效果。

D3.js的坑

性能问题

使用前预估过可能发生的性能问题，但是还是高估了D3.js的性能处理，或者说网页本身作为展示平台的性能瓶颈，实测大概绘制5000个点，网页就自动崩溃。

审美问题

如photoshop这样工具，我基本可以熟练操作，但是每逢调色，配色这样的问题，只能束手无策。数据可视化不仅仅是代码部分，而且要有“最佳视觉” 这个素养并不是临时能培养起来的，过于底层的东西，在操作便利性上的提升就导致了提高了操作者的要求。

学习资源

由于我前面说的原因，真正写底层可视化的人并不多，在国内相应的学习资料较少，而且陈旧，目前D3.js已经升级到v4版本，大多资源还停留在v3版本。同样在英语世界，资料也并不多，有时候遇到bug，找起原因来简直令人崩溃。
