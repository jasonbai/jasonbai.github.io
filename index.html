数据可视化感悟

发表于 2017-07-02

成功提交

终于算是成功通过了Udacity-数据分析的P6-可视化大作业。结束了将近一个月的折磨。

项目背景

这次课程的主题是可视化，作业开头的导言写的很好。

为什么进行这个项目？
该项目涉及对于有效数据可视化至关重要的总体态度和信念，如：

  • 可视化是一场对话
  • 向别人展示和分享可视化
  • 可视化是一个流动过程，通常需要多次改进迭代

你将有机会体验端对端的有效数据可视化创建过程，突出原本隐藏或很难发现的重要数据信息。

我会学到什么？
完成此项目后，你将能够：

  • 展现出选择最佳视觉元素来编码数据和批判性评估可视化效果的能力
  • 使用交互式可视化向合适的受众讲述故事或发现
  • 经历迭代式可视化创建流程和使用 dimple.js 或 d3.js 建立交互式可视化。

在日常的工作中，没有数据分析师给我提供分析报告，我自己边做分析，边写结论，很多都是用Excel或者RStudio直接出图，接着继续寻找自己感兴趣的东西。没有在意太多需要对外沟通的东西。如题中所说，可视化是一场对话，过去我是自己和自己对话，正因如此，这次作业花了很大的精力去做，与自己对话，与他人对话都是应该去训练的。

这次可视化作业，我没有直接使用D3.js，而是用dimple.js来制作，主要原因：懒。

D3.js偏向底层，有大量的可定制化的细节自由调整。可是除去专业人士需要那样疯狂的调整细节，多数人如我一样，手头就有很好的工具，简单的如excel，高级的Tableau，阿里云提供的datav，都要比纯粹的写D3.js要更快且更有效率。当然，由于作业要求，起码要使用dimple.js这样的封装了部分D3.js代码的工具来完成。

既然最终呈现，是通过网页展示，利用前期学的一点点网站前端的经验，套用了一个html5的模板，进行了css代码的微调，最终呈现出了http://www.jasonbai.com/Udacity-P6 的效果。

D3.js的坑

性能问题

使用前预估过可能发生的性能问题，但是还是高估了D3.js的性能处理，或者说网页本身作为展示平台的性能瓶颈，实测大概绘制5000个点，网页就自动崩溃。

审美问题

如photoshop这样工具，我基本可以熟练操作，但是每逢调色，配色这样的问题，只能束手无策。数据可视化不仅仅是代码部分，而且要有“最佳视觉” 这个素养并不是临时能培养起来的，过于底层的东西，在操作便利性上的提升就导致了提高了操作者的要求。

学习资源

由于我前面说的原因，真正写底层可视化的人并不多，在国内相应的学习资料较少，而且陈旧，目前D3.js已经升级到v4版本，大多资源还停留在v3版本。同样在英语世界，资料也并不多，有时候遇到bug，找起原因来简直令人崩溃。

用R语言解读逻辑回归

发表于 2017-04-09

一、为什么用逻辑回归模型？

先谈线性回归，是利用数理统计中回归分析，来确定两种或两种以上变量间相互依赖的定量关系的一种统计分析方法，运用十分广泛。其表达形式为y = w’x+e，e为误差服从均值为0的正态分布。

线性回归模型一般是对某一事物进行预测，例如预测房价、身高、GDP、学生成绩等，这些被预测的变量都属于连续型变量。

但是有些情况下，被预测变量可能是二元变量

即成功或失败、流失或不流失、涨或跌等，对于这类问题，线性回归将束手无策。

这个时候就需要另一种回归方法进行预测，即Logistic回归。

二、逻辑回归的主要三大用途

  1. 寻找危险因素，找到某些影响因变量的”坏因素”，一般可以通过优势比发现危险因素；

  2. 用于预测，可以预测某种情况发生的概率或可能性大小；

  3. 用于判别，判断某个新样本所属的类别。

三、与普通线性回归的关系

  1. Logistic回归模型的因变量为二分类变量；

  2. 该模型的因变量和自变量之间不存在线性关系；

  3. 一般线性回归模型中需要假设独立同分布、方差齐性等，而Logistic回归模型不需要；

  4. Logistic回归没有关于自变量分布的假设条件，可以是连续变量、离散变量和虚拟变量；

  5. 由于因变量和自变量之间不存在线性关系，所以参数(偏回归系数)使用最大似然估计法计算。

四、逻辑回归的数学解释

Logistic回归主要通过构造一个重要的指标：发生比来判定因变量的类别。在这里我们引入概率的概念，把事件发生定义为Y=1，事件未发生定义为Y=0，那么事件发生的概率为p,事件未发生的概率为1-p，把p看成x的线性函数；

回归中，最常用的估计是最小二乘估计，因为使得p在[0,1]之间变换，最小二乘估计不太合适，有木有一种估计法能让p在趋近与0和1的时候变换缓慢一些（不敏感），这种变换是我们想要的，于是引入Logit变换,对p/(1-p)也就是发生与不发生的比值取对数，也称对数差异比。经过变换后，p对x就不是线性关系了。

Drools介绍

Drools（JBoss Rules ）具有一个易于访问企业策略、易于调整以及易于管理的开源业务规则引擎，符合业内标准，速度快、效率高。业务分析师或审核人员可以利用它轻松查看业务规则，从而检验是否已编码的规则执行了所需的业务规则。

官方网站：http://www.drools.org/

环境设置

我的本机系统环境 MacOS 10.11.6，本机运行Drools Workbench需要有Tomcat和MySQL，个人电脑没有配置，需要从头搭建环境。

安装Tomcat

安装Tomcat，参考文章

  1. 启动服务：打开终端， cd /Users/apple/Library/Tomcat/bin
  2. 再输入： ./startup.sh
  3. 网址输入 http://localhost:8080/
  4. 关闭服务：终端输入： ./shutdown.sh

  1. 配置完成后验证，在终端输入：

mysql -u root -p

《描述性统计学》笔记

发表于 2017-02-08

今天开始学习描述性统计学入门课程，随堂进行笔记记录。

1.什么是统计学？

统计学是数学的一个重要领域，用于分析、解释和预测数据的结果。描述统计学将让你学会描述数据的基本概念。这是为那些对数据科学、经济学、心理学、机器学习、体育分析学以及其它任何领域感兴趣的人开设的意义重大的入门课程。本课程提供中文版本。

2.什么是描述性统计？

描述性统计是指将调查样本中的包含的大量数据资料进行整理、概况和计算。是推断性统计的基础。

2.1 主要运用工具

集中量数，如平均数、中位数、众数、几何平均数、调和平均数。

变异量数，如全距、平均差、标准差、相对差、四分差。

常见的描述性统计方法可分为三类：

  • 用数据的统计量来描述，如：均值、标准差等；
  • 用图示技术来描述，如：直方图、散布图、趋势图、排列图、条形图和饼分图等；
  • 用文字语言分析和描述，如：统计分析表、分层、因果图、亲和图和流程图等。
