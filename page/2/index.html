绑定个人域名到Github Page（Hexo版）

发表于 2016-07-23

在依照“使用 Hexo 为自己在 Github 上建一个静态 Blog 站点” 后，进一步优化站点。

目前做了以下升级：

  • 通过修改theme的CSS，调整了部分样式
  • 绑定个人域名，替换Github提供的二级域名

本文简单介绍一下绑定二级域名的过程。

注册个人域名，且修改CNAME指向

在域名服务提供商注册域名后，修改记录类型CNAME,记录值为现有的Github的二级域名（例如我个人的jasonbai.github.io）

添加CNAME文件

在~/source/ 文件下添加文件名为CNAME文件，文件内容添加个人域名地址（例如我个人的www.jasonbai.com）

上传CNAME


hexo generate


hexo deploy

便携Wiki软件TiddlyWiki简明使用手册

发表于 2016-07-21

由于需要多个文档进行管理，且需要及时更新并公布给使用者。寻找到了TiddlyWiki这款软件。

一个基于 JavaScript 全交互维基。它可以作为浏览器中的单一HTML文件,高度可定制.

满足了目前的需求

  • 可以快速传播，U盘拷贝，网络传输，无需搭建环境
  • 高度定制，便捷的文档管理及索引
  • 支持插件，简单的WikiText语言
  • 支持中文，不需要任何特殊知识或工具

1.安装

中文官网：http://www.tiddlywiki.cn/

依照官网的GettingStarted进行,下载保存空的TiddlyWiki。

Tips:

  • 最好采用Chrome下载保存，Safari有可能会卡死
  • 不要更改扩展名

2.设置

在下载后，直接开个文件，进行设置。如果没有中文语言包，在设置中找到插件标签，进行插件安装。

TiddlyWiki支持Markdown插件，经过试用编辑起来bug较多，建议试用WikiText进行文档输入编辑。

WikiText语法

WikiText（维基语法）是一种简洁的、富于表现力的、支持广泛的文本格式、超文本和交互功能的输入方法。它可以让你专注于写作而不是提供复杂的用户界面。其设计亲近 MarkDown语法 的用户，但更关注链接和互动功能。

2.1 WikiText 常用语法

块引用


<<<
This is a block quoted paragraph
written in English
<<<

显示为

This is a block quoted paragraph
written in English

标题


! This is a level 1 heading


!! This is a level 2 heading


!!! This is a level 3 heading

前置输入!, ‘！’越多，标题字体越小

列表

列表控制由#和*进行有序列表、无序列表设置

3.保存和传输

TiddlyWiki在页面点击保存，在浏览器上保存后自行命名保存。注意采用相同命名后会替换原有版本。建议在为了版本管理，采用命名升级的方式。单一文件可以通过u盘拷贝复制，也可以通过网络传输。

4.上传展示

TiddlyWiki提供http://tiddlyspot.com 站点服务，可以将自己的wiki上传展示，个人大概看了这个站点，速度较慢，体验一般，并没有深入使用。

另外一种方式是丢在GitHub上面，如果有更新可以随时通过git的方式进行上传，但注意上传Github后wiki为开放状态，注意保护个人隐私。
将文件名命名为index.html ，使用github page服务进行展示的教程在这里：Creating Project Pages manually
